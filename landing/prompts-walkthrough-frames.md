# Care One OS — Walkthrough Frame Prompts (A/B Split)

**Purpose:** Generate 16 photorealistic images (2 per scene) for the "A Day in the Life with Care One OS" scroll-based walkthrough.

**Total Frames:** 16 (8 scenes x 2 angles each)
- **Frame A — The Human Moment:** Wide/medium shot of Sarah in a caring interaction. No device in focus. Emotional anchor.
- **Frame B — The Tech Moment:** Medium/close-up of Sarah using a device. App screenshot popups will be composited on top afterwards.

**Aspect Ratio:** 16:9 (1920x1080) landscape — optimised for horizontal scroll viewport.

**Scroll Behaviour:** As the user scrolls through each scene, Frame A crossfades into Frame B — showing the human side first, then the technology that supports it.

## Generation Workflow

### Step 1: Generate the Reference Image
Generate **Frame 1A** first. This is your character anchor. Review it carefully:
- Does Sarah match the description? (Hair, outfit, age, expression)
- Is the setting right? (Warm home, not clinical)
- Is the quality/style what you want?

If yes, use this image as the **character reference** for all other frames.

### Step 2: Generate All Frames Using Character Reference
- **Flux:** Use the `--cref` flag or IP-Adapter with Frame 1A as reference
- **Leonardo.ai:** Upload Frame 1A as character reference image
- **Midjourney:** Use `--cref [Frame 1A URL]` flag
- **ComfyUI + SDXL:** Use IP-Adapter FaceID with Frame 1A as the face reference
- **Seedance / video AI:** Generate a single consistent video, extract frames

Generate in order: 1B, 2A, 2B, 3A, 3B ... 8A, 8B. Check each against Frame 1A for consistency before moving on.

### Step 3: Post-Processing (Compositing App Screenshots)
For every **B frame**:
1. Identify where the device is in the image
2. Place 2-3 real Care One OS app screenshots as floating popups near the device:
   - **Primary popup:** Large, ~40% of frame width, slight 2-3 degree rotation, positioned above/beside the device
   - **Secondary popup:** Smaller, ~25% of frame width, opposite side, 5 degree rotation the other way
   - **Optional tertiary:** Small badge/notification, ~15% width
3. Apply to each popup:
   - Rounded corners: 16px
   - Drop shadow: `0 20px 60px rgba(0,0,0,0.25)`
   - Subtle border: `1px solid rgba(255,255,255,0.15)`
   - Soft outer glow in the SOMA colour for that feature (teal for dashboard, orange for medication, etc.)
4. Add a slight gaussian blur to the popup edges to blend with the photographic background
5. Match the popup lighting to the frame's lighting (warmer popups for evening scenes, brighter for morning)

### Step 4: Final Exports
- Export each composited frame at **1920x1080** (16:9) for web
- Also export at **3840x2160** (4K) if the source images support it
- Save both with and without composited popups (the raw frames are useful for the crossfade animation)
